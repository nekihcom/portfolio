## mainブランチをプッシュする際の運用ルール

以下の手順を厳守してください。特に `本番環境にデプロイして` と指示された場合は、ステップを順番通りに実施します。

1. `dev` ブランチに未コミットファイルがないことを確認する。
2. `dev` ブランチでプッシュを実施する。
3. `dev` ブランチでビルドを実行する。
   - ビルドエラーが発生した場合  
     1. エラー箇所と原因、想定する解決策を報告する。  
     2. ユーザーの承認を得てから解決策を実行する。  
     3. 再ビルドし、エラーが再発した場合は再度報告からやり直す。
4. ビルドが成功したらコミットする。
5. 再度 `dev` ブランチでプッシュする。
6. `dev` ブランチを `main` ブランチへマージする。
7. `main` ブランチをプッシュする。
8. 作業ブランチを `dev` に戻す。

上記手順は例外なく必ず遵守してください。
