# ポートフォリオサイト要件定義書

## 概要

個人のスキルや制作実績、アウトプットを発信することを目的としたポートフォリオサイトを構築する。  
本サイトは、企業の採用担当者やクライアントに向けた自己紹介、およびブログ発信・制作物紹介を主な機能とし、問い合わせ機能および管理者向けの編集機能も備える。

---

## 1. 機能一覧

### 1.1 フロントエンド（ユーザー向けページ）

| ページ名     | 説明 |
|--------------|------|
| トップページ | サイト全体の概要、最新記事や作品のピックアップ表示 |
| 自己紹介     | プロフィール、経歴、スキルなどを掲載 |
| ブログ一覧   | 投稿されたブログ記事の一覧を表示 |
| ブログ詳細   | 各記事の本文やメタ情報を表示 |
| 作品一覧     | 制作物のサムネイル・概要を一覧表示 |
| 作品詳細     | 各作品の詳細情報、画像、リンクなどを表示 |
| お問い合わせ | フォームからメッセージを送信（メール or DB保存） |

### 1.2 バックエンド（管理者向け機能）

| 機能名       | 説明 |
|--------------|------|
| ブログ作成   | 新規ブログ記事を投稿する機能 |
| ブログ編集   | 既存のブログ記事を編集・更新する機能 |
| 作品作成     | 新規制作物を登録する機能 |
| 作品編集     | 既存の制作物を編集・更新する機能 |

---

## 2. 技術スタック

### 2.1 フロントエンド

- **Next.js（App Router）**  
  Reactベースのフレームワークで、静的生成（SSG）・サーバーサイドレンダリング（SSR）・インクリメンタル静的再生成（ISR）に柔軟に対応できる。Vercelとの親和性も高く、スケーラブルな構成を容易に実現できる。

- **TypeScript**  
  型安全性を担保することで、開発初期から保守性・信頼性の高いコードベースを構築可能。バグの早期発見やドキュメントの補完にも効果的。

- **shadcn/ui**  
  Tailwind CSS + Radix UIベースの高品質コンポーネント群を提供。カスタマイズが容易で、デザインに統一感を持たせやすく、開発効率も高い。

- **Storybook**  
  UIコンポーネントの開発・ドキュメント化ツール。非エンジニアともデザインの確認ができ、再利用性や保守性の高いUI設計に役立つ。

---

### 2.2 バックエンド

- **Django + Django REST Framework**（推奨）  
  Python製の高機能Webフレームワーク。管理画面や認証機能、REST APIの構築が容易で、少ないコードで堅牢なシステムを構築できる。Supabaseとの連携や、JWTを用いたAPI認証にも対応しやすいため、開発と運用のバランスが取れている。

---

### 2.3 データベース / 認証 / ストレージ

- **Supabase**  
  Firebaseの代替として注目されるオープンソースのBaaS。PostgreSQLベースのDBに加え、認証（Auth）、ストレージ、Edge Functionsを備え、バックエンド構築の負担を大幅に軽減できる。ポートフォリオレベルのアプリケーションに最適。

---

### 2.4 デプロイ / CI/CD

- **Vercel**（フロントエンド用）  
  Next.jsとの親和性が高く、GitHubとの連携で自動デプロイが可能。CDNによる高速配信、ISR対応、プレビュー機能など、モダンなWeb開発環境が整っている。

- **GitHub Actions**  
  テスト、型チェック、Storybookのビルド、デプロイなどCI/CDパイプラインを構築するためのGitHub公式ツール。運用コストが低く、保守性も高い。

## 3. レンダリング方式（Next.js）

| ページ / 機能         | レンダリング方式 | 理由 |
|------------------------|------------------|------|
| トップページ           | SSG              | 静的な構成で頻繁に更新されない |
| 自己紹介               | SSG              | 同上 |
| ブログ一覧             | ISR              | 投稿追加に対応するため定期的に再生成 |
| ブログ詳細             | ISR              | 各記事ごとの再生成が可能で高速な表示 |
| 作品一覧               | ISR              | 新規作品追加に対応するため |
| 作品詳細               | ISR              | 同上 |
| お問い合わせ           | SSG + CSR         | 表示は静的、フォーム送信はクライアントサイドで処理 |
| 管理機能（作成・編集） | SSR or SPA       | 認証が必要で、最新の状態を即時反映させたい |

---

## 4. 認証とセキュリティ

- Supabase Auth によるユーザー認証を導入する。
- 管理画面や投稿系APIは認証ユーザーに限定する。
- API通信にはJWT等を用いた認可処理を行う。

---

## 5. リポジトリ構成

本プロジェクトは**モノレポ（単一リポジトリ）構成**を採用する。  
開発効率と見通しの良さを重視し、フロントエンドとバックエンドを1つのリポジトリで管理する。

- ルートディレクトリ：`/portfolio`
- フロントエンド：`/portfolio/frontend`
- バックエンド：`/portfolio/backend`

---

### ディレクトリ構成例

/portfolio  
├── /frontend # Next.js App Router (TypeScript)  
│ ├── /app  
│ ├── /components  
│ ├── /lib  
│ ├── /stories # Storybook関連  
│ └── ...  
├── /backend # Django + DRF  
│ ├── /portfolio_api  
│ ├── /apps  
│ ├── /migrations  
│ └── manage.py  
├── /infra # Docker設定や.env管理など（任意）  
├── README.md  
└── .github/workflows # GitHub ActionsでCI/CD管理  


---

## 6. 開発・運用フロー

1. GitHubリポジトリにてブランチ運用（例：`main` / `develop`）
2. Pull Request作成 → GitHub Actionsで Lint / Type Check / Storybook build を実行
3. マージ後、自動デプロイ（フロントエンドはVercel、バックエンドは別ホスティング）
4. Supabase 管理画面または `/admin` 機能からコンテンツ管理

---

## おわりに

この要件定義書は、ポートフォリオサイトの構築にあたって、設計・実装を担当するエンジニアやAIエージェントがスムーズに理解・開発できるよう作成されたものである。  
将来的な拡張や改善にも柔軟に対応できるよう、モダンかつ実用性の高い構成を採用している。
